尝试绕过TracerPID反调试

准备工作

Windows下Linux环境准备

1. 安装[MinGW](http://www.mingw.org/), 在官网下好之后, 右键按下图选中要下载的，之后直接在`Installation`中直接选中`Apply change`即可。

    ![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552553170/aa_catmea.png)
2. 进入安装目录`<MinGW安装的地方>\MinGW\bin`, 将`gcc.exe`重命名为`cc.exe`, 就短暂地这样改就好了，之后用完make工具再改回来。反正我没找到哪里可以改Makefile的内容。不这样修改的话会出现如下报错：

    ![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552554377/boerror_ltdeqw.png)

    具体原因好像是MinGW的bug, 在Linux环境下一般gcc是等同于cc的, 然而在Windows中却不是这样的, 如果有人找到Makefile的内容可以在哪里改的话, 可以告诉我=v=, 虽然下面并没有可以评论的地方。

    *注：还有千万别学人家瞎改名, 把`mingw-make.exe`改成`make.exe`直接用, 这样也会报错！*

准备boot

  1. 提取boot.img 

    

输入以上命令即可找到boot的位置。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552572141/boposition_jqxant.png)

将boot导出成boot.img。这里一定要注意, 要按照你自己的情况将boot导出, 要修改`if=/dev/block/[实际boot文件夹名]`。



![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552640930/bocorrect_whui56.png)

重新开一个命令行页面，或直接`exit`退出，输入如下指令：
`adb root pull /data/local/boot.img boot.img`

这里加了一个root是因为之前直接pull的时候说我权限不够。加了root之后发现还是一直失败, 就尝试先将boot.img存到`/sdcard/Download`下面之后直接用pull指令就成功了。


在哪个目录下pull的，文件就会存在哪个目录下。

  2. 准备好bootimg-tools工具

    `git clone https://github.com/pbatard/bootimg-tools.git`

下载后打开文件夹到`[MinGW安装的目录]]\MinGW\msys\1.0`, 双击`msys.bat`即可进入到一个模拟Linux环境的GUN界面中。进入`[bootimg-tools所在文件夹]/bootimg-tools`(如果找不到的话可以用everything找找看), 用`make`指令生成二进制文件。

![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552574870/afmake_l4lqwi.png)

将boot.img拷到`[bootimg-tools所在文件夹]\bootimg-tools\mkbootimg`文件夹下, 执行命令`./unmkbootimg -i boot.img`。(要记录下面输出的一大段字，之后要重新生成boot.img的时候直接复制修改成相应kernel文件名和输出文件名即可)
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552641959/succesun_umfiqv.png)

  3. 提取原始zImage

    复制`kernel`文件并将其重命名为`zImage.gz`。用`010 Editor`工具找到16进制`1f 8b 08 00`， 删掉查找到的位置的前面的所有数据, 保存之后再执行命令`gunzip zImage.gz`。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552650755/succesch_yoj1nf.png)

到这里zImage文件就提取完成了。

    修改内核

    说明

    我这里用的是IDA 7.0

    正式开始

  4. 用IDA直接打开zImage文件, 并做如下改动

    ![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552653260/IDA1_zugiws.png)

点击OK之后, 继续按照下面修改。

  5. 定位函数位置

    打开命令提示符, 输入如下指令关闭符号屏蔽(符号屏蔽又是个啥？？？)并查看函数地址, 因为在IDA中查看二进制文件不好直接用函数名进行定位。


函数地址如下：
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552655198/fuposition_lrj63p.png)

在IDA中按下快捷键G跳转到c01b0884(__task_pid_nr_ns函数)
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552659250/f1_otrhpf.png

之后点击函数名c01b0884(__task_pid_nr_ns函数), 按下快捷键X, 会弹出交叉引用, 在其中查找我们上面找的第二个函数c02ba04c( proc_pid_status)的位置。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552659761/f2_iuuo1l.png)

好的, 上面选中的并不是我想找的函数TAT。这个时候, 就在IDA界面中按下`Shift+F12`, 在字符串界面查找`TracerPid`。找到之后双击查询后的结果。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552661104/f3_zrynlg.png)

IDA在解析的时候没有解析好这个文件, 出现了下面的情况。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552661543/f4_tn3sq3.png)

这时候选中`S`的位置即`0x53`按下`A`, 即可正常解析。如果解析出来还不是下面这种格式，可以鼠标左键选中该字符串的所有内容再按一次`A`。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552703909/CHSU_cre1cq.png)
如果字符串都解析出来之后, 还是识别不出来c02ba04c( proc_pid_status)函数的话, 直接`G`跳转到c02ba04c, 并按下C, 但是识别的函数也不是正确的。所以这里采用第三篇参考文章的方法, 直接修改TracePid字符串。

  1. 修改TracePid字符串

    (1) 选中`TracePid`中的`%d`
    ![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552706938/S1_g980jo.png)

(2) 将`TracePid`中的`%d`改为`0\t`, 选中`%d`后按下`Alt`+`4`。
    ![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552707440/p1_mebj3m.png)

上图中第四行第一个`%d`就是我们要修改的(TracerPid后的字符串), 十六进制为`25 64`。点击`25`前面按下`F2`, 输入`30 09`, 再按下`F2`就修改成功了。
    ![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552708361/p2_luiaxd.png)

最后保存修改的内容, 按下图点开之后直接点OK即可。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552708651/p3_zozpyy.png)

  2. 修改kernel文件

    在MinGW命令行界面输入命令`gzip -n -f -9 zImage`, 压缩修改好的文件`zImage`生成`zImage.gz`。用`010 Editor`打开文件`kernel`, 并在`kernel`中查找`1F 8B 08 00`。(下图是覆盖完成的样子)
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1553071178/1553071092_1_fg4s6b.png)

选中`1F 8B 08 00`位置之后，点开`Edit`->`Insert/Overwrite`->`Overwrite File...`, 选择`zImage.gz`文件即可, 这样就不用自己计算文件大小并且直接覆盖了, 完成之后点击保存为新的文件`boot-new.img`。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552722307/succ_eyefab.png)

使用执行unmkbootimg生成的命令, 修改-o参数名(要输出的文件名), 直接拷贝到MinGW命令行中直接执行。

  3. 刷机

    我这里用的是[Nexus 5](http://www.kgfanr.com/articles/3019.html), 直接在关机状态下按`音量键-`和`开机键`(或输入命令`adb reboot bootloader`)即可进入fastboot状态。在命令行中输入`fastboot flash boot boot-new.img`, 之后再输入`fastboot reboot`即可。
![](https://res.cloudinary.com/dq4c0zqqy/image/upload/v1552722558/succ2_yzvx0w.png)

刷完倒是能正常开机了, 但是我手机又要重新root了.....(这个时候有尝试过用SuperSU刷成砖, 还是用magisk+twrp比较好)
